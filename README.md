# Приложение на React Native с картой и локальной базой данных – Задание 2
Продолжая работу над предыдущим заданием, вам предстоит расширить функциональность приложения, добавив поддержку локальной базы данных с использованием SQLite и реализовав управление состоянием с помощью React Context. Это задание фокусируется на сохранении данных и правильной архитектуре приложения.

# Инструкции по установке

1. Клонируйте репозиторий:
```
   git clone ...
   cd ...
```
Или скачайте ZIP архив:
- Зайдите на GitHub репозиторий
- Нажмите "Code" → "Download ZIP"
- Распакуйте архив
2. Установите зависимости:
```
   npm install --legacy-peer-deps

```
   
3. Запустите проект через Expo:
```
 npm expo start
```
   
4. Откройте приложение в Expo Go на вашем устройстве.
Отсканируйте QR-код в терминале:
- Android: Откройте Expo Go → "Scan QR Code"
- iOS: Отсканируйте камерой, откроется Expo Go

# Документация схемы базы данных

- В проекте используется SQLite для локального хранения данных маркеров и изображений.

## Таблицы:
- Markers
  - id INTEGER PRIMARY KEY AUTOINCREMENT
  - latitude REAL NOT NULL,
  - longitude REAL NOT NULL,
  - created_at DATETIME DEFAULT CURRENT_TIMESTAMP

- Marker_Images
  - id INTEGER PRIMARY KEY AUTOINCREMENT
  - marker_id INTEGER NOT NULL
  - uri TEXT NOT NULL
  - created_at DATETIME DEFAULT CURRENT_TIMESTAMP
  - FOREIGN KEY (marker_id) REFERENCES markers (id) ON DELETE CASCADE

# Описание подхода к обработке ошибок

- Все запросы к БД предварительно проверяют факт инициализации базы данных, и только после этого выполняются.
- Ошибки навигации и работы с изображениями также обрабатываются соответствующим образом.
- При иных проблемах доступа к базе данных приложение информирует пользователя и предотвращает дальнейшее использование функционала, связанного с хранением данных.
